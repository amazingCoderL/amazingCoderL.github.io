RxJava学习笔记

发表于 2017-07-27

特点

  1. 异步
  2. 简洁（代码逻辑简洁），链式调用

RxJava的观察者模式

  1. Observable(被观察者)
  2. Observer(观察者)
  3. subscribe(订阅)
  4. Event(事件)：
    • 包括onNext(),onCompleted(),onError()
    • onCompleted()指事件队列完结，即不会有新的onNext()发出时触发
    • onError()事件处理异常，事件队列终止

基本实现

一. 创建Observer

  1. 代码实现

  2. Subscribe实现

    实现自Observer的抽象类，实现方法与Observer一样，Subscribe的不同点：

    • onStart():事件还未发送前调用，做一些初始化操作。
    • unsubscribe():取消订阅调用，s事件队列完成后必须做unsubscribe()操作，防止内存泄露。
二. 创建Observable

  1. 代码实现

    一定要理解这句话:

    OnSubscribe 会被存储在返回的 Observable 对象中，它的作用相当于一个计划表，当 Observable 被订阅的时候，OnSubscribe 的 call() 方法会自动被调用

  2. 其他实现方式

    • just(T…): 将传入的参数依次发送出来。
    • from(T[]) / from(Iterable<? extends T>) : 将传入的数组或 Iterable 拆分成具体对象后，依次发送出来。
三. Subscribe(订阅)

  1. 代码实现

    注意代码实现方式，被观察者订阅观察者的实现顺序。

  2. 订阅做了哪些事

    • 调用 Subscriber.onStart() 。
    • 调用 Observable 中的 OnSubscribe.call(Subscriber) 。
    • 将传入的 Subscriber 作为 Subscription 返回。

  3. Action0()和Action1()实现不完整定义回调

线程控制– Scheduler(一)

  1. Scheduler的API使用场景

    • Schedulers.immediate():默认Schedulers，不指定线程；
    • Schedulers.newThread():开启新线程
    • Schedulers.io():开启I/O操作，其实是开启无数量上限的线程池
    • Schedulers.computation():开启固定的线程池

  2. 控制线程API

    • subscribeOn():控制事件订阅线程
    • observeOn():控制事件消费的线程
